# 뉴스피드 "최근 주요 뉴스" + 상시 필터 툴바 설계

## 배경

직전 작업([[2026-06-20-newsfeed-today-design]])에서 `src/app/news/page.tsx`에 "오늘의 뉴스"(KST 기준 오늘 발행분만 필터)와 "전체보기" 토글(클릭해야 검색/정렬/필터 툴바와 전체 목록이 펼쳐지는 구조)을 추가했다. 그런데 "오늘"만 보여주면 오늘 발행된 뉴스가 없는 날에는 섹션이 비어 사용성이 떨어지고, 필터/검색 기능이 토글 뒤에 숨어 있어 페이지 전체에 적용되지 않는 문제가 있다.

## 목표

- 상단 하이라이트를 "오늘 발행된 뉴스"가 아니라 "심각도 + 최신순 기준 주요 뉴스"로 교체한다 — 오늘 뉴스가 없는 날에도 항상 의미 있는 콘텐츠를 보여준다.
- 검색/정렬/필터 툴바를 "전체보기" 토글 뒤에 가두지 않고 페이지 상단에 항상 노출해 전체 페이지에 적용되도록 한다.

## "최근 주요 뉴스" 선정 기준

- 검색/필터 상태와 **독립적으로** 전체 `news` 배열에서 계산한다(사용자가 검색어를 입력해도 이 섹션은 바뀌지 않음).
- 정렬 기준: 심각도(`SEVERITY_RANK`: critical → info) 우선, 동일 심각도 내에서는 `published_at` 최신순.
- 상위 6개를 표시한다.

```ts
const highlightNews = useMemo(
  () =>
    [...news]
      .sort((a, b) => {
        const rankDiff =
          (SEVERITY_RANK[a.severity.toLowerCase()] ?? 5) -
          (SEVERITY_RANK[b.severity.toLowerCase()] ?? 5);
        if (rankDiff !== 0) return rankDiff;
        return new Date(b.publishedAt).getTime() - new Date(a.publishedAt).getTime();
      })
      .slice(0, 6),
  [news]
);
```

## 컴포넌트 구조 변경 (`src/app/news/page.tsx`)

1. **Hero 섹션** — 제목 "오늘의 보안 뉴스" → "최신 보안 뉴스"로 변경. 날짜 표시 라인은 그대로 유지.
2. **필터 툴바** — 기존에 "전체보기" 패널 내부에 있던 검색/정렬/필터/뷰토글 sticky 툴바를 Hero 섹션 바로 아래(현재 "오늘의 뉴스" 섹션이 있던 위치)로 이동한다. 항상 렌더링한다.
3. **최근 주요 뉴스 섹션** (기존 "오늘의 뉴스" 섹션 교체)
   - 섹션 제목: "오늘의 뉴스" → "최근 주요 뉴스"
   - 새로고침 버튼: 기존과 동일하게 유지 (`handleRefresh`, 전체 뉴스 재조회)
   - `loading`: `SkeletonCard` 6개 (기존 3개에서 표시 개수에 맞춰 조정)
   - `error`: 기존과 동일한 에러 메시지 블록 유지
   - `news.length === 0` (로딩 완료, 에러 없음): 안내문을 "오늘은 아직 수집된 뉴스가 없습니다." → "아직 수집된 뉴스가 없습니다."로 변경
   - 그 외: `highlightNews`를 기존과 동일한 3열 그리드(`NewsCard`)로 표시, 클릭 시 `setSelectedItem`
4. **심각도 범례 + 기사 목록(`filtered`)** — 기존 "전체보기 패널" 안에 있던 심각도 범례와 검색/필터/정렬이 적용된 전체 목록(그리드/리스트 토글 포함)을 항상 렌더링한다. 로직(`filtered` 계산, 뷰 모드 전환)은 변경 없음.
5. **제거 대상**: `expanded` state, "전체보기" 토글 버튼, 이를 감싸던 `{expanded && (...)}` 조건 렌더링 — 모두 삭제.

## 부수 정리

- `isTodayKST`의 유일한 사용처가 사라지므로 `src/lib/date.ts` 파일 자체를 삭제한다. (다른 import 없음, 별도 테스트 파일 없음 — 확인됨)

## 변경하지 않는 것

- `filtered` 계산 로직(검색/소스/직군 필터, 정렬) — 그대로 유지
- `NewsCard`, `NewsListItem`, `SkeletonCard`, `NewsDetailModal` 등 기존 컴포넌트 — 변경 없음
- 뷰 토글(grid/list), `/api/news` 호출 — 변경 없음

## 테스트 계획

- `highlightNews` 정렬 로직 확인: critical 1건 + info 3건이 혼재할 때 critical이 항상 상단에 오는지, 동일 심각도 내에서는 최신순인지 확인
- `news.length === 0`일 때 "아직 수집된 뉴스가 없습니다." 안내문이 노출되는지 확인
- 필터 툴바가 페이지 로드 시 항상 보이는지(토글 없이) 확인
- 새로고침 버튼 클릭 시 `fetch`가 재호출되는지 확인 (기존 동작 유지, mock fetch)

## 범위 제외

- 서버 측 정렬/필터링(쿼리 파라미터 추가 등)은 하지 않는다 — 클라이언트 측 파생으로 충분
- "최근 주요 뉴스"에 대한 별도 캐싱이나 폴링은 추가하지 않는다

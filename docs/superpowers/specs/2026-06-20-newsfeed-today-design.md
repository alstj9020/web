# 뉴스피드 "오늘의 뉴스" + 전체보기 설계

## 배경

`src/app/news/page.tsx`는 헤더에 "오늘의 보안 뉴스"라고 표시하지만 실제로는 `published_at` 기준 필터링 없이 전체 뉴스를 심각도→날짜 순으로 한 번에 보여준다. 오늘 발행된 뉴스를 상단에 강조하고, 나머지(전체) 뉴스는 별도 "전체보기" 영역으로 분리한다.

## 목표

- DB의 `published_at`을 KST 캘린더 날짜 기준으로 비교해 "오늘" 뉴스를 상단에 표시
- 상단 "오늘의 뉴스"는 새로고침 버튼으로 즉시 재조회 가능
- 그 아래 "전체보기" 토글 영역에 기존 검색/정렬/필터/뷰 전환 기능을 유지한 전체 뉴스 목록 표시 (오늘 뉴스 중첩 포함)

## "오늘" 날짜 기준

- KST(Asia/Seoul) 캘린더 날짜로 비교한다. 서버는 UTC로 데이터를 보관하므로, 비교 시 `Intl.DateTimeFormat("ko-KR", { timeZone: "Asia/Seoul" })` 등으로 `published_at`과 현재 시각을 모두 KST `YYYY-MM-DD` 문자열로 변환 후 동일 여부로 판단한다.
- 신규 유틸 함수 `isTodayKST(isoString: string): boolean`을 `src/lib/date.ts`(신규 파일)에 추가한다.

## 데이터 흐름

- 기존 `/api/news` 라우트(`revalidate = 300`, 5분 ISR)는 변경하지 않는다.
- 클라이언트는 기존과 동일하게 마운트 시 `/api/news`를 호출해 전체 뉴스를 받아온다.
- `todayNews = useMemo(() => news.filter(n => isTodayKST(n.publishedAt)), [news])`로 파생한다. 별도 API 호출이나 서버 필터링은 하지 않는다.
- 새로고침 버튼 클릭 시 fetch 로직을 재실행하는 `refetch()`를 호출한다. ISR 캐시가 만료되지 않았다면 동일한 데이터가 반환될 수 있음을 사용자가 인지하고 있다 — 별도 안내 문구는 추가하지 않는다.

## 컴포넌트 구조 변경

`src/app/news/page.tsx` 내 JSX 구조를 다음과 같이 재구성한다 (로직은 그대로, 신규 state만 추가):

1. **Hero 섹션** — 기존 그대로 유지 (제목 "오늘의 보안 뉴스" + 날짜)
2. **오늘의 뉴스 섹션** (신규)
   - 섹션 헤더 + 새로고침 버튼(아이콘 버튼, 로딩 중 스피너)
   - `loading`: `SkeletonCard` 3개 (기존 컴포넌트 재사용)
   - `error`: 기존과 동일한 에러 메시지 표시
   - `todayNews.length === 0` (로딩 완료, 에러 없음): "오늘은 아직 수집된 뉴스가 없습니다" 안내문
   - 그 외: `NewsCard` 3열 그리드 (기존 그리드 스타일 재사용), 클릭 시 기존과 동일하게 `setSelectedItem`
3. **전체보기 토글 버튼** (신규)
   - `expanded` state (`useState(false)`, 기본 닫힘)
   - 버튼 레이블: `전체보기 (${news.length}건)`, chevron 아이콘이 펼침 상태에 따라 회전
4. **전체보기 패널** (`expanded === true`일 때만 렌더)
   - 기존 툴바(검색/정렬/필터/뷰토글) + 심각도 범례 + 그리드/리스트 목록을 그대로 이동
   - 기존 `filtered`(검색/정렬/필터 적용된 `news` 전체 기준) 로직은 변경 없음 — 오늘 뉴스도 포함된 채로 표시됨(중첩 허용)
   - `expanded === false`일 때는 이 블록 전체를 렌더하지 않음(언마운트)

## 변경 파일 목록 (예상)

- `src/lib/date.ts` (신규) — `isTodayKST` 유틸
- `src/app/news/page.tsx` — 오늘의 뉴스 섹션, 새로고침 버튼, 전체보기 토글 추가. 기존 툴바+목록 블록은 전체보기 패널 안으로 이동
- 신규 컴포넌트는 만들지 않는다 (`NewsCard`, `SkeletonCard`, `NewsListItem` 등 기존 컴포넌트 그대로 재사용)

## 테스트 계획

- `isTodayKST` 유닛 테스트: UTC 자정 경계값(예: KST 09:00 = UTC 00:00)에서 날짜가 올바르게 갈리는지 확인
- 오늘 뉴스 0건일 때 안내문이 노출되는지 확인
- 전체보기 토글 펼침/접힘 시 패널이 마운트/언마운트되는지 확인
- 새로고침 버튼 클릭 시 `fetch`가 재호출되는지 확인 (mock fetch)

## 범위 제외

- 서버 측 날짜 필터링(쿼리 파라미터 추가 등)은 하지 않는다 — 클라이언트 측 파생으로 충분
- 폴링(자동 주기 재조회)은 추가하지 않는다 — 수동 새로고침 버튼으로 충분
